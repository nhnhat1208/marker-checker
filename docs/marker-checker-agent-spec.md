# Marker Checker Agent Doc Map

The documentation is separated into three groups:

1. product spec and requirements
2. technical design and implementation decisions
3. delivery planning and review notes

## Group 1: Product Spec

Purpose:

- define what the product does
- define workflow and user-facing behavior
- define request and audit requirements

Use this group when:

- defining requirements
- reviewing workflow behavior
- clarifying request and audit expectations

Documents:

- [Product Spec Index](./product-spec/README.md)
- [Overview](./product-spec/overview.md)
- [Workflow And Lifecycle](./product-spec/workflow-and-lifecycle.md)
- [Data Model And Audit](./product-spec/data-model-and-audit.md)
- [Chat And Review Capabilities](./product-spec/chat-and-review-capabilities.md)

## Group 2: Technical Design

Purpose:

- define how the initial release should be built
- document architecture and channel decisions
- document the recommended tech stack

Use this group when:

- deciding architecture
- choosing channel or integration approach
- reviewing implementation tradeoffs

Documents:

- [Technical Design Index](./technical-design/README.md)
- [Architecture](./technical-design/architecture.md)
- [Configuration And Integrations](./technical-design/configuration-and-integrations.md)
- [Scope And Channel Decision](./technical-design/scope-and-channel-decision.md)
- [Recommended Tech Stack](./technical-design/recommended-tech-stack.md)

## Group 3: Delivery Planning

Purpose:

- define initial release scope and milestones
- identify risks and assumptions
- capture review questions before implementation starts

Use this group when:

- planning the build
- reviewing scope cuts
- checking open questions and delivery risks

Documents:

- [Delivery Planning Index](./delivery-plan/README.md)
- [Implementation Plan](./delivery-plan/implementation-plan.md)
- [Risks, Assumptions, And Open Questions](./delivery-plan/risks-assumptions-and-open-questions.md)

## Suggested Reading Order

1. Product overview
2. Workflow and lifecycle
3. Data model and audit
4. Technical architecture
5. Scope and channel decision
6. Recommended tech stack
7. Implementation plan
8. Risks, assumptions, and open questions
